### How Crompressor Works

1. **Train** a codebook from your data (one-time cost)
2. **Pack** splits data into chunks, finds the closest codebook pattern via LSH, stores only the XOR delta
3. **Unpack** reconstructs the original data by applying the delta to the codebook pattern
4. The delta pool is further compressed with Zstandard for maximum density

### When Crompressor Excels

- Structured/repetitive data (source code, configs, logs, JSON)
- Domain-specific data where a trained codebook captures the patterns
- Low-entropy data with polynomial patterns (fractal O(1) engine)

### When to Use Standard Tools

- Already-compressed data (images, videos, archives)
- Purely random data (entropy > 7.8)
- One-off compression without a training set
